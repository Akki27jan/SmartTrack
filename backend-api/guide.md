## ID Format

All IDs are auto-generated by the server during signup in the format **`AAT000`**:

| Part | Meaning | Example |
|------|---------|---------|
| `AA` | First 2 letters of the name (uppercased) | `TE` for "TechNova" |
| `T`  | Type character — `C` (Company), `E` (Employee), `H` (HR) | `C` |
| `000`| Incrementing number starting from `001` | `001` |

**Example:** The first company named "TechNova" gets the ID `TEC001`.

> **Note:** Users do NOT provide the ID — it is returned in the signup response.

## Signup Endpoints

### 1. Company Signup

**`POST /company/signup`**

Registers a new company and returns a generated Company ID.

**Request Body:**

| Field      | Type   | Required | Description                  |
|------------|--------|----------|------------------------------|
| `Name`     | string | ✅       | Company name                 |
| `Gstin`    | string | ✅       | GSTIN number (must be unique)|
| `Address`  | string | ✅       | Company address              |
| `Password` | string | ✅       | Account password             |
| `Domain`   | string | ✅       | Business domain / industry   |

**Example Request:**

```json
{
    "Name": "TechNova Solutions",
    "Gstin": "27AABCT1234F1ZH",
    "Address": "42 MG Road, Pune, Maharashtra",
    "Password": "techNova@2026",
    "Domain": "IT Services"
}
```

**Success Response (`200`):**

```json
{
    "message": "Company registered successfully",
    "Company_ID": "TEC001"
}
```

**Error Responses:**

| Status | Detail |
|--------|--------|
| `400`  | `"A company with this GSTIN already exists"` |

### 2. Employee Signup

**`POST /employee/signup`**

Registers a new employee and returns a generated Employee ID.

**Request Body:**

| Field      | Type   | Required | Description                    |
|------------|--------|----------|--------------------------------|
| `Name`     | string | ✅       | Employee full name             |
| `Email`    | string | ✅       | Email address (must be unique) |
| `Password` | string | ✅       | Account password               |

**Example Request:**

```json
{
    "Name": "Rahul Sharma",
    "Email": "rahul.sharma@gmail.com",
    "Password": "rahul@Secure1"
}
```

**Success Response (`200`):**

```json
{
    "message": "Employee registered successfully",
    "Employee_ID": "RAE001"
}
```

**Error Responses:**

| Status | Detail |
|--------|--------|
| `400`  | `"An employee with this email already exists"` |

---

### 3. HR Signup

**`POST /hr/signup`**

Registers a new HR representative linked to an existing company.

**Request Body:**

| Field           | Type   | Required | Description                            |
|-----------------|--------|----------|----------------------------------------|
| `Company_ID`    | string | ✅       | ID of the company (must already exist) |
| `Name`          | string | ✅       | HR person's full name                  |
| `Company_Email` | string | ✅       | Company email address (must be unique) |
| `Password`      | string | ✅       | Account password                       |

**Example Request:**

```json
{
    "Company_ID": "TEC001",
    "Name": "Anjali Verma",
    "Company_Email": "anjali.verma@technova.com",
    "Password": "anjaliHR@01"
}
```

**Success Response (`200`):**

```json
{
    "message": "HR registered successfully",
    "HR_ID": "ANH001"
}
```

**Error Responses:**

| Status | Detail |
|--------|--------|
| `400`  | `"Company Does Not Exist"` — the provided `Company_ID` is not in the database |
| `400`  | `"An HR with this company email already exists"` |

## Login Endpoints

> **Important:** Login only works after a successful signup. The credentials used during signup are the same ones used to log in.

---

### 4. Company Login

**`POST /company/login`**

Authenticates a company using GSTIN and password.

**Request Body:**

| Field      | Type   | Required | Description          |
|------------|--------|----------|----------------------|
| `Gstin`    | string | ✅       | Registered GSTIN     |
| `Password` | string | ✅       | Account password     |

**Example Request:**

```json
{
    "Gstin": "27AABCT1234F1ZH",
    "Password": "techNova@2026"
}
```

**Success Response (`200`):**

```json
{
    "message": "Company login successful",
    "Company_ID": "TEC001",
    "Name": "TechNova Solutions"
}
```

**Error Responses:**

| Status | Detail |
|--------|--------|
| `401`  | `"Invalid GSTIN or password"` |

---
